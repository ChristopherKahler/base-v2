<objective>
## Goal
Stand up a compiling Rust binary (`base`) with embedded Oxigraph that can load the `ops:` vocabulary from a TTL file, parse TriG workspace graphs, run SPARQL queries against them, and write results back atomically. This is the foundation everything else targets.

## Purpose
Phase 0 proves the core data path: TTL file → in-memory Oxigraph → SPARQL query → atomic write-back. Every subsequent phase (hooks, CRUD, extraction, signals) depends on this pipeline working correctly. If this is wrong, everything built on it is wrong.

## Output
- Compiling Rust binary with `base --version` and `base --help`
- `ops:` vocabulary TTL defining all classes and predicates
- Store module: load TriG → query → atomic write-back
- Unit tests proving the round-trip
</objective>

## Architecture Reference
ARCHITECTURE.md §2 (locked decisions), §3 (persistence spine), §7 (graph-neighborhood vision) define the constraints this plan implements against.

## Key Constraints From Architecture
- TTL-is-the-store (Flavor B) — no separate database, TriG text files
- Every write = short-lived process: load → SPARQL UPDATE → atomic write-back (temp + rename)
- Every read = load file(s) into in-memory Oxigraph, query, exit
- IRI scheme: path/slug-based stable identity, never generated ids
- Named graphs: `ops:graph/global` (`.base-gbl/graph.trig`), `ops:graph/ws/{slug}` (`{workspace}/.base/graph.trig`)
</context>

<task type="auto">
  <name>Task 2: ops: vocabulary + store module</name>
  <files>src/ontology/mod.rs, src/ontology/ops.ttl, src/store.rs</files>
  <action>
    Define the ops: vocabulary as Turtle in src/ontology/ops.ttl:

    **Namespace:** `@prefix ops: <http://ops-sys.local/ontology#> .`

    **Classes (rdfs:Class):**
    - ops:Project (subtypes: ops:App, ops:Framework, ops:TrackingProject)
    - ops:Task
    - ops:Decision
    - ops:Entity (subtypes: ops:Person, ops:Organization)
    - ops:Goal
    - ops:Reminder
    - ops:Workspace
    - ops:Domain
    - ops:Rule
    - ops:Document
    - ops:PaulProject

    **Core predicates (define domain + range via rdfs):**
    - ops:name (→ xsd:string)
    - ops:description (→ xsd:string)
    - ops:status (→ xsd:string) — values: active, completed, blocked, deferred, superseded, archived
    - ops:priority (→ xsd:string) — values: urgent, high, medium, low
    - ops:lastActive (→ xsd:dateTime)
    - ops:createdAt (→ xsd:dateTime)
    - ops:updatedAt (→ xsd:dateTime)
    - ops:supersedes (→ owl:Thing) — links to prior version
    - ops:supersededBy (→ owl:Thing)
    - ops:belongsTo (→ ops:Workspace) — entity → workspace edge
    - ops:hasDomain (→ ops:Domain) — project/workspace → domain
    - ops:hasRule (→ ops:Rule) — domain → rule
    - ops:hasGoal (→ ops:Goal)
    - ops:blockedBy (→ xsd:string) — free-text blocker description
    - ops:nextAction (→ xsd:string)
    - ops:revenue (→ xsd:string)
    - ops:dueDate (→ xsd:date)
    - ops:staleThreshold (→ xsd:integer) — days before considered stale

    **IRI scheme convention (documented in TTL comments):**
    - Projects: `ops:project/{slug}` (e.g., `ops:project/casegate-v2`)
    - Tasks: `ops:task/{project-slug}/{task-slug}`
    - Decisions: `ops:decision/{domain}/{slug}`
    - Entities: `ops:entity/{slug}`
    - Workspaces: `ops:workspace/{slug}`
    - Domains: `ops:domain/{slug}`
    - Rules: `ops:rule/{domain-slug}/{N}`
    - Goals: `ops:goal/{slug}`

    Create src/store.rs with:
    - `pub fn load_graph(path: &Path) -> Result<Store>` — reads a TriG file, parses into in-memory Oxigraph Store
    - `pub fn load_graphs(paths: &[&Path]) -> Result<Store>` — loads multiple TriG files into one store (cross-tier query)
    - `pub fn query(store: &Store, sparql: &str) -> Result<QueryResults>` — runs SPARQL SELECT/ASK
    - `pub fn write_back(store: &Store, path: &Path) -> Result<()>` — serializes store to TriG, writes to temp file, renames atomically
    - All errors use a project-level error type (anyhow or thiserror)

    Create src/ontology/mod.rs:
    - `pub const OPS_TTL: &str = include_str!("ops.ttl");` — embeds vocabulary at compile time
    - `pub fn load_vocabulary(store: &Store) -> Result<()>` — parses OPS_TTL into the store's default graph
  </action>
  <verify>cargo build (store module compiles, vocabulary embeds)</verify>
  <done>AC-2 satisfied: ops.ttl defines all classes/predicates; store module provides load/query/write-back</done>
