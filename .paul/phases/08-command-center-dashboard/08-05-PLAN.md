<objective>
## Goal
Upgrade all 4 dashboard panels from read-only displays into interactive operator surfaces — graph hot-reload, task/entity creation, domain rules viewer, context bracket display, usage time-range picker, hook event log rotation, and data export.

## Purpose
The 4 panels are live but mostly passive. This plan makes the dashboard a working tool: create tasks without leaving the browser, inspect domain rules visually, see which context bracket the current session is in, control the usage time window, and export data for sharing. These are the features that make someone want to keep the dashboard open.

## Output
- Graph hot-reload button + auto-detect
- Task creation form in Operations panel
- Entity creation form in Graph Explorer
- Domain rules viewer (sidebar or modal)
- Live context bracket display in Session Activity header
- Usage time-range picker (7d/30d/90d)
- Hook event log rotation (auto-truncate)
- Export buttons (graph JSON, usage CSV)
</objective>

<task type="auto">
  <name>Task 1: Backend endpoints — reload, create, domains, time-range, export, log rotation</name>
  <files>src/dashboard/api.rs, src/dashboard/server.rs</files>
  <action>
    **Graph reload:**
    - POST `/api/graph/reload` — re-read graph.trig into the Mutex Store, replacing current data
    - Return `{"reloaded": true, "nodes": N, "edges": N}`

    **Task creation:**
    - POST `/api/ops/task` — accept `{"name": "...", "project": "...", "status": "active"}`
    - SPARQL INSERT: create Task entity with IRI, name, status, hasTask edge from project
    - Write back to trig

    **Entity creation:**
    - POST `/api/graph/entity` — accept `{"name": "...", "type": "...", "domain": "..."}`
    - SPARQL INSERT: create entity with IRI, type, name, hasDomain edge
    - Write back to trig

    **Domain rules:**
    - GET `/api/domains` — read domains.toml (workspace + global), return JSON array
    - Each domain: `{"name": "...", "mode": "...", "prompt_keywords": [...], "paths": [...], "rules": [...]}`

    **Usage time-range:**
    - Modify `parse_usage_data` to accept `days: u32` parameter
    - GET `/api/usage/summary?days=7` (default 30)
    - GET `/api/usage/sessions?days=7` (default 30)

    **Export:**
    - GET `/api/export/usage-csv` — return CSV with columns: date, project, model, input, output, cost
    - GET `/api/export/graph-json` — return full graph as JSON (nodes + edges, same shape as existing endpoints)
    - Set Content-Disposition header for download

    **Log rotation:**
    - On dashboard startup: if hook-events.jsonl > 10MB, truncate to last 5000 lines
    - Simple: read file, keep tail, write back

    **PAUL document auto-linking (graph hygiene):**
    - In `src/extract/frontmatter.rs`: detect `.paul/` in file path
    - Auto-create `relatedTo` edge to the project entity (derive from paul.json or workspace slug)
    - Auto-create `relatedTo` edges between PLAN → SUMMARY pairs (same phase/plan number)
    - This fixes orphaned PAUL nodes floating disconnected in the graph

    **Routes:** Wire all new endpoints in server.rs
  </action>
  <verify>
    - `cargo test` passes
    - `cargo build --release` succeeds
    - Manual: `curl -X POST http://localhost:3741/api/graph/reload` returns JSON
    - Manual: `curl http://localhost:3741/api/domains` returns domain list
  </verify>
  <done>AC-1 server-side, AC-2 server-side, AC-3 server-side, AC-4 server-side.</done>
