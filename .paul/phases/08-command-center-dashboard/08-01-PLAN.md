<objective>
## Goal
Stand up the embedded dashboard server and ship the Graph Explorer panel — a D3.js force-directed visualization of the knowledge graph served by the base binary itself via `base dashboard`.

## Purpose
The visual surface that makes BASE tangible and marketable. Screenshots of this dashboard are content. The graph explorer turns "RDF knowledge graph" from an abstract concept into a thing you can see, click, and navigate.

## Output
- `base dashboard` command starts axum server on localhost, opens browser
- SPARQL-backed JSON API endpoints for graph data
- Svelte SPA with dark-theme Graph Explorer panel (D3.js force-directed)
- Frontend compiled to static assets, embedded in binary via include_dir!
- Build script (`build.rs`) that runs `npm run build` in dashboard/ before cargo compile
</objective>

<task type="auto">
  <name>Task 2: SPARQL-backed Graph API endpoints</name>
  <files>src/dashboard/api.rs</files>
  <action>
    Create src/dashboard/api.rs with axum route handlers:

    GET /api/graph/nodes — return all entities from the workspace graph:
    - Run SPARQL: SELECT ?s ?type ?name WHERE { GRAPH ?g { ?s rdf:type ?type . OPTIONAL { ?s {p}:name ?name } } }
    - Return JSON array: [{ iri, type, name, ... }]
    - Include entity metadata: status, path, documentType, tags (aggregate hasTag values)

    GET /api/graph/edges — return all relationships:
    - Run SPARQL: SELECT ?s ?p ?o WHERE { GRAPH ?g { ?s ?p ?o } } with predicate filter
    - Filter to meaningful predicates (relatedTo, references, hasRule, hasDecision, hasMilestone, hasTask, calls, importsFrom, contains)
    - Skip metadata predicates (lastExtracted, path, name — those are node properties, not edges)
    - Return JSON array: [{ source, target, predicate }]

    GET /api/graph/node/:iri — return single node detail:
    - All triples where ?s = iri (outgoing) and ?o = iri (incoming)
    - Return JSON: { iri, type, name, properties: {}, outgoing: [], incoming: [] }

    GET /api/graph/search?q=term — search nodes by name:
    - SPARQL FILTER(CONTAINS(LCASE(?name), LCASE(?q)))
    - Return matching nodes array

    All endpoints load the workspace graph from cwd/.base/graph.trig.
    Return proper JSON Content-Type. Empty graph = empty arrays, not 500s.
    Share the graph store across requests via axum State (load once at server start).
  </action>
  <verify>
    Start server, curl http://localhost:3741/api/graph/nodes returns JSON array.
    curl /api/graph/edges returns edges array.
    curl /api/graph/search?q=project returns matching nodes.
    Empty workspace returns [].
  </verify>
  <done>AC-2 satisfied: API returns real graph data as JSON</done>
</task>
