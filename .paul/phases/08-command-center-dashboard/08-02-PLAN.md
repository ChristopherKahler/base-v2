Ship the Operations panel (kanban + table views of projects/milestones/tasks) and OperatorNotes (first write mutation — annotate any graph entity from the dashboard).

- Operations panel with kanban and table views

- People, decisions, reminders displayed in operations view

  <name>Task 1: OperatorNotes API + graph persistence</name>
  <files>src/dashboard/api.rs, src/dashboard/server.rs</files>
  <action>
    Add POST /api/graph/node/:iri/notes endpoint:
    - Accept JSON body: { "text": "note content" }
    - Query existing notes for this entity to determine next index
    - Create OperatorNote entity: rdf:type ops:OperatorNote, ops:text, ops:index, ops:createdAt
    - Create edge: entity ops:operatorNote note_iri
    - Write graph back to disk (persist)
    - Return the created note as JSON: { index, text, createdAt }

    Add GET /api/graph/node/:iri/notes endpoint:
    - Query all OperatorNote entities linked to the given IRI
    - Return sorted by index: [{ index, text, createdAt }]

    Update node_detail endpoint to include notes in response.

    The graph store needs to support writes — currently loaded read-only into Arc.
    Change AppState.store to Arc<RwLock<Store>> or load-mutate-writeback per request.
    Simplest: use std::sync::Mutex around the store for write operations, since
    writes are rare (only notes) and reads don't need true concurrency at this scale.

    Wire new routes in server.rs router.
  </action>
