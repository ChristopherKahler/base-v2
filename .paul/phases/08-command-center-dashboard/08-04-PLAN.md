<objective>
## Goal
Add Usage Analytics panel that parses Claude Code's JSONL session files to show token usage, cost tracking, model distribution, and session breakdowns. Remove debug logging. Final panel — completes the Command Center Dashboard.

## Purpose
Usage Analytics closes the visibility loop. Operators can see how much Claude Code costs, which models burn the most tokens, which sessions are expensive, and how usage trends over time. Combined with Graph Explorer (what the graph knows), Operations (what's being worked on), and Session Activity (what hooks are doing), the four panels give complete operational visibility into a Claude Code deployment.

## Output
- `UsageAnalytics.svelte` — charts and tables for token/cost analysis
- `/api/usage/summary` endpoint — parses JSONL, returns aggregated data
- `/api/usage/sessions` endpoint — per-session breakdowns
- Debug `eprintln!` removed from WS handler
</objective>

<task type="auto">
  <name>Task 1: Usage API endpoints + JSONL parsing</name>
  <files>src/dashboard/api.rs, src/dashboard/server.rs</files>
  <action>
    **JSONL parsing:**
    - Discover session files: walk `~/.claude/projects/` for `*.jsonl` files
    - For each file, read lines, find entries with `message.usage` field
    - Extract per-message: model, input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens, timestamp (from parent message)
    - Skip non-message entries (type: "last-prompt", "mode", "summary", etc.)
    - Derive project name from directory path: `-home-user-workspace-project` → last segment

    **Cost estimation:**
    - Hardcode current model pricing (per 1M tokens):
      - claude-opus-4* — input: $15, output: $75, cache_read: $1.50, cache_write: $18.75
      - claude-sonnet-4* — input: $3, output: $15, cache_read: $0.30, cache_write: $3.75
      - claude-haiku-4* — input: $0.80, output: $4, cache_read: $0.08, cache_write: $1
    - Match model string with starts_with for version flexibility

    **GET /api/usage/summary:**
    - Returns JSON:
      ```json
      {
        "total_input": N,
        "total_output": N,
        "total_cache_read": N,
        "total_cache_write": N,
        "estimated_cost_usd": N.NN,
        "session_count": N,
        "models": {"claude-opus-4-6": {"input": N, "output": N, "cost": N.NN}, ...},
        "daily": [{"date": "2026-06-02", "input": N, "output": N, "cost": N.NN}, ...]
      }
      ```
    - Cache the parsed result in AppState (behind Mutex) — re-parse only if oldest cache is >60s old
    - Limit to last 30 days of data to avoid parsing 528MB on every load

    **GET /api/usage/sessions:**
    - Returns JSON array of session summaries:
      ```json
      [{"session_id": "...", "project": "...", "model": "...", "input": N, "output": N, "cost": N.NN, "started": "ISO", "messages": N}, ...]
      ```
    - Sorted newest-first, limit 50 sessions

    **Routes:**
    - Add both routes to server.rs

    **Cleanup:**
    - Remove the 3 `eprintln!("[ws] ...")` debug lines from the WebSocket handler

    Avoid: Parsing all 414 sessions (528MB) on every request — use the 30-day window + caching.
    Avoid: External dependencies for JSONL parsing — serde_json line-by-line is sufficient.
  </action>
