<objective>
## Goal
Create the `manifest.toml` system that tracks installed ChrisAI components, versions, activation status, and update check state. Integrate manifest writes into the existing `base install` flow. Add `base activate <key>` command for Skool members.

## Purpose
The manifest is the single source of truth for what's installed. Every downstream phase depends on it — update checks (Phase 11) read it, `base update` (Phase 12) compares against it, unofficial detection (Phase 13) checks for its absence.

## Key decisions
- **One static activation key** — hardcoded in Skool classroom. Not per-user.
- **`base activate <key>`** — validates locally against compiled secret, writes token to manifest. No network call.
- **Version checks via npm registry + GitHub API** — no custom chrisai.cv endpoint needed.
- **Product name: ChrisAI** — npm package `chrisai`, attribution references Chris AI Systems.

## Output
- New `src/manifest.rs` module with structs, load/save, token validation, component detection
- New `base activate <key>` command
- Updated `base install` to write manifest.toml after every install
- New `--full` flag on `base install` to register all 4 components
</objective>

<task type="auto">
  <name>Task 1: Create manifest module</name>
  <files>src/manifest.rs, src/lib.rs</files>
  <action>
    Create `src/manifest.rs` with:

    **Structs (all Serialize + Deserialize):**
    - `Manifest` — top-level container
      - `chrisai: ChrisAI`
      - `components: HashMap<String, ComponentEntry>`
      - `update_check: UpdateCheck`
    - `ChrisAI`
      - `installed_at: String` (ISO 8601)
      - `source: String` (default "https://chrisai.cv/skool")
      - `token: String` (default empty — activation key from Skool classroom)
    - `ComponentEntry`
      - `version: String`
      - `path: String`
      - `installed_at: String` (ISO 8601)
    - `UpdateCheck`
      - `last_checked: String` (ISO 8601)
      - `ttl_seconds: u64` (default 604800 — 7 days)
      - `pending_update: String` (default empty)
      - `dismissed_until: String` (default empty)
    
    **Version check sources (no custom endpoint):**
    - npm packages: `https://registry.npmjs.org/paul-framework/latest` → `.version`
    - npm packages: `https://registry.npmjs.org/@chrisai/seed/latest` → `.version`
    - npm packages: `https://registry.npmjs.org/@chrisai/skillsmith/latest` → `.version`
    - BASE binary: `https://api.github.com/repos/ChristopherKahler/base/releases/latest` → `.tag_name`

    **Use `serde` rename attributes:**
    - Top-level table names: `chrisai` via `#[serde(rename = "chrisai")]`
    - `update_check` → `update_check` (underscore is fine in TOML)

    **Functions:**
    - `Manifest::load() -> Option<Manifest>` — read from `~/.base-gbl/manifest.toml`, return None if missing or unparseable
    - `Manifest::save(&self) -> Result<()>` — atomic write (temp + rename) to `~/.base-gbl/manifest.toml`, follow the existing pattern from config.rs
    - `Manifest::manifest_path() -> Option<PathBuf>` — resolve `~/.base-gbl/manifest.toml`
    - `Manifest::is_activated(&self) -> bool` — compare `token` against the one compiled-in static key. Token IS the key (not HMAC-derived). One key, static, hardcoded in Skool classroom. Return `true` if `self.chrisai.token == ACTIVATION_KEY`.
    - `detect_component(name: &str) -> Option<ComponentEntry>` — scan known paths for a component:
      - "base": check `~/.local/bin/base` exists, version from `env!("CARGO_PKG_VERSION")`
      - "paul": check `~/.claude/paul-framework/` exists, read version from package.json
      - "seed": check `~/.claude/commands/seed/` exists, read version from package.json
      - "skillsmith": check `~/.claude/commands/skillsmith/` exists, read version from package.json

    **Activation key:** One static string compiled into the binary (placeholder for now, replaced before release). NOT HMAC — just a string comparison. The key is the secret. Distribute it in Skool classroom only.

    Register `pub mod manifest;` in `src/lib.rs`.
  </action>
  <verify>

    Create a simple test: construct a Manifest, serialize to TOML string, deserialize back, assert fields match.
  </verify>
