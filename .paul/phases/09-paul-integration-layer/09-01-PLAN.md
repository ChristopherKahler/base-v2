<objective>
## Goal
Build the ledger.toml extractor and session-cost join pipeline — BASE v2's side of the PAUL cost attribution system. When PAUL produces `ledger.toml` entries (action/phase/plan/timestamp), BASE extracts them into the graph and joins against Claude Code session JSONL to attribute token costs to specific PAUL actions.

## Purpose
Today BASE v2 knows session costs (from JSONL parsing) and knows PAUL project structure (from paul.toml extraction), but can't answer "how much did Phase 2 cost?" or "what's the lifetime build cost of this project?" The ledger bridges that gap — PAUL logs what it did and when, BASE figures out what that cost.

## Output
- `ledger.toml` file spec documented
- New `src/extract/ledger.rs` extractor producing `ops:LedgerEntry` triples
- Timestamp-to-session matching logic in dashboard API
- `/api/ops/ledger` endpoint returning entries enriched with cost data
- `/api/ops/cost-summary` endpoint returning per-phase and per-milestone cost rollups
</objective>

## Ledger Spec (from PAUL v1.4 coordination)
```toml
# .paul/ledger.toml — append-only session history
[[entry]]
action = "plan"       # plan | apply | unify | iteration | verify | transition | milestone_create | milestone_complete
phase = 2
plan = "02-01"
at = "2026-06-03T10:30:00-05:00"
note = ""             # optional — why an iteration happened
```

## How the join works
1. BASE extracts ledger.toml → ops:LedgerEntry triples in graph (action, phase, plan, timestamp)
2. BASE already parses session JSONL with collect_all_events() → Vec<UsageEvent>
3. Join: for each ledger entry, find the session whose time range contains the entry's `at` timestamp
4. Result: ledger entry + session tokens/cost = per-action cost attribution
