<objective>
## Goal
Build a Cost Attribution panel that consumes the `/api/ops/ledger` and `/api/ops/cost-summary` endpoints to display per-phase, per-action cost breakdowns for PAUL-tracked projects.

## Purpose
Plan 09-01 shipped the extraction pipeline and API layer. The data exists but has no visual surface. This plan closes the loop — operators can see exactly how much each phase/action costs, identify expensive operations, and track project lifetime spend. This is what makes PAUL cost attribution tangible and marketable.

## Output
- New `CostAttribution.svelte` component with summary cards, phase breakdown chart, and action drill-down table
- Integration into Usage Analytics panel via sub-navigation toggle
</objective>

<task type="auto">
  <name>Task 1: Create CostAttribution.svelte component</name>
  <files>dashboard/src/panels/CostAttribution.svelte</files>
  <action>
    Create a new Svelte component that fetches and displays PAUL cost attribution data.

    **Data loading:**
    - Call `getCostSummary()` on mount, store result in `summary` reactive variable
    - Call `getLedger()` on mount, store in `ledger` for the event log section
    - Show loading skeleton while fetching, empty state if no data

    **Summary cards row (top):**
    - Total Project Cost (formatted as $X.XX)
    - Total Ledger Entries (count)
    - Most Expensive Phase (name + cost)
    - Average Cost Per Entry (total / count)
    - Use the same card pattern as UsageAnalytics stat cards (dark card, gradient border, label + value)

    **Phase breakdown chart (middle):**
    - Horizontal bar chart using D3.js (already available in the project)
    - Each bar = one phase from `summary.phases[]`
    - Bar width proportional to `phase.total_cost`
    - Label: phase name (left), cost + session count (right)
    - Sorted by total_cost descending
    - Bars use Huly design system gradient fills (not flat colors)
    - Clickable — sets `selectedPhase` reactive variable

    **Action drill-down (below chart, conditional):**
    - Visible only when `selectedPhase` is set
    - Table with columns: Action | Cost | Sessions | Avg Cost/Session
    - Data from `selectedPhase.actions[]`
    - Table follows existing OperationsPanel table styling (dark rows, subtle borders)
    - Click phase bar again or click "×" to collapse

    **Ledger event log (bottom):**
    - Chronological list of recent ledger entries from `getLedger()` (most recent first, cap at 20)
    - Each row: timestamp (relative, e.g. "2h ago"), action badge, phase, plan, note
    - Action badges color-coded: plan=blue, apply=green, unify=purple, other=gray
    - Compact rows, no expand/collapse needed

    **Styling:**
    - Match Huly design system: dark canvas #090A0C background, gradient depth over shadows
    - CSS scoped within component (Svelte default)
    - Responsive: stack cards vertically on narrow viewports
    - Use CSS custom properties already defined in the dashboard (--primary, --green, --accent-purple, etc.)
  </action>
  <verify>Component imports without errors, renders with mock/real data when mounted</verify>
  <done>AC-1 satisfied: summary cards display totals. AC-2 satisfied: phase bars render proportionally. AC-3 satisfied: clicking phase expands action table.</done>
</task>
