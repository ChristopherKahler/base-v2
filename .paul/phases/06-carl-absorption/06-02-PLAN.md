<objective>
## Goal
Port DEVMODE dashboard and context brackets from Python CARL into the Rust binary. DEVMODE emits a structured telemetry block instructing Claude what to report. Context brackets track session depth (prompt count) and adjust injection behavior — lean on fresh sessions, force-refresh on deep ones.

## Purpose
DEVMODE is Chris's primary observability tool for tuning the context system — it shows what domains matched, what was injected, what was skipped. Context brackets prevent firehose injection on fresh sessions and ensure stale-context recovery on deep sessions. Both are core operational features of the unified BASE tool.

## Output
- `[devmode]` config section in base.toml with `enabled` flag
- `[context_bracket]` config section with threshold tuning
- Session state tracks `prompt_count` and derives bracket (FRESH/MODERATE/DEPLETED/CRITICAL)
- user_prompt_submit emits bracket tag + DEVMODE instruction block alongside rules
</objective>

    **src/config.rs** — Add BracketConfig to BaseConfig:
    ```
    pub struct BracketConfig {
        pub fresh_until: u32,      // default 3
        pub moderate_until: u32,   // default 10
        pub depleted_until: u32,   // default 20
        pub refresh_interval: u32, // default 5 (force re-inject every Nth prompt in DEPLETED+)
        pub enabled: bool,         // default true
    }
    ```
    Add `pub bracket: BracketConfig` to BaseConfig with `#[serde(default)]`.

    **src/domain/session.rs** — Add prompt tracking:
    - Add `prompt_count: u32` field to SessionState (with `#[serde(default)]`)
    - Add `pub fn increment_prompt(&mut self) -> u32` — increments and returns new count
    - Add `pub fn bracket(&self, config: &BracketConfig) -> Bracket` — derives bracket from prompt_count
    - Add enum `Bracket { Fresh, Moderate, Depleted, Critical }` with Display impl
    - Add `pub fn should_force_refresh(&self, config: &BracketConfig) -> bool` — true if DEPLETED+ AND prompt_count % refresh_interval == 0

    **src/hook/user_prompt_submit.rs** — Bracket-aware injection:
    - Call `session.increment_prompt()` early in handle()
    - Derive bracket from session state
    - Emit bracket tag at top of output: `<context-bracket>[FRESH]</context-bracket>` (or similar)
    - In FRESH mode (first 2 prompts): skip neighborhood context, inject rules only
    - In DEPLETED+: call `should_force_refresh()` — if true, clear dedup for all domains this prompt (force re-inject)
    - Pass bracket info to DEVMODE block (Task 2)

    **tests/context_bracket_test.rs** — Tests:

    - Test: FRESH mode skips neighborhood
    - Test: DEPLETED force-refresh overrides dedup
    - Test: bracket state persists across prompts in session

    Avoid: Changing the matching logic. Brackets affect OUTPUT behavior, not matching.
    Avoid: Making brackets affect session-start signals (those are separate).
