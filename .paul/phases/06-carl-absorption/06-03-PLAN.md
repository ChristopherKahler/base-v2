<objective>
## Goal
`base learn` — graph-backed structured memory. Notes are entities with typed edges to domains, projects, and people. `base recall` queries relationally, not by flat keyword search. Notes surface automatically alongside domain rules when their linked domain matches.

## Purpose
Replaces PSMM and Claude's native `/memory`. The whole point of an ontological graph is that memories aren't isolated — they know their neighbors. A note linked to CaseGate surfaces when CaseGate's domain matches. A note about auth surfaces when auth-related files are edited. Memory becomes relational.

## Output
- `ops:Note` class in ontology with typed predicates
- `base learn` CLI command (creates Note entity with edges)
- `base recall` CLI command (SPARQL-backed relational search)
- Auto-surface: notes linked to matched domains appear in user_prompt_submit injection
</objective>

    - `ops:noteType` (string) — insight, correction, decision, commitment, shift
    - `ops:relatedTo` (general) — edge from Note to any entity (domain, project, entity, etc.)

    **src/crud/note.rs** — New module:
    - `pub fn learn(cwd, ns, text, domain, note_type, project, entity) -> Result<String>`
      - Build IRI: ops:note/{slug} (slug from text)
      - INSERT: rdf:type ops:Note, noteText, noteType, createdAt, status "active"
      - If domain: add ops:relatedTo edge to ops:domain/{domain_slug}
      - If project: add ops:relatedTo edge to ops:project/{project_slug}
      - If entity: add ops:relatedTo edge to ops:entity/{entity_slug}
      - Return slug

    - `pub fn recall(cwd, ns, keyword, domain) -> Result<()>`
      - SPARQL SELECT: notes matching keyword in text OR linked to domain
      - If keyword: FILTER(CONTAINS(LCASE(?text), keyword))
      - If domain: UNION with notes linked to ops:domain/{slug} via ops:relatedTo
      - Print table: type | text | linked-to | created

    **src/crud/mod.rs** — Add `pub mod note;`

    **src/cli.rs** — Wire commands:
    - `base learn --text "..." --domain X --type insight` (domain and type optional, type defaults to "insight")
    - `base learn --text "..." --project Y --type correction`
    - `base recall --keyword "..." --domain X` (both optional, at least one required)
